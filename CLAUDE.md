# 지뢰찾기 (Minesweeper) 웹 게임

## 프로젝트 개요
브라우저에서 동작하는 지뢰찾기 게임. 빌드 도구 없이 순수 HTML/CSS/JavaScript로 제작하여
파일을 열거나 간단한 정적 서버만 띄우면 바로 실행 가능해야 한다.

## 기술 스택
- HTML5 / CSS3 / Vanilla JavaScript (ES Modules)
- 빌드 도구, 프레임워크, 외부 라이브러리 사용하지 않음
- 저장은 브라우저 `localStorage` 사용 (최고 기록)

## 파일 구조
```
Minesweeper/
├── index.html          # 진입점, 게임판/헤더/모달 마크업
├── css/
│   └── style.css        # 전체 스타일 (다크모드 포함)
├── js/
│   ├── main.js           # 앱 초기화, 이벤트 바인딩
│   ├── board.js           # 보드 생성, 지뢰 배치, 인접 지뢰 수 계산
│   ├── game.js            # 게임 상태 관리 (진행/승리/패배), 오픈/플래그 로직
│   ├── timer.js           # 타이머 시작/정지/리셋
│   └── storage.js         # localStorage 최고 기록 read/write
└── CLAUDE.md
```

## 게임 규칙 및 기능 범위
### 기본 기능
- 난이도 선택: 초급(9x9, 지뢰 10), 중급(16x16, 지뢰 40), 고급(16x30, 지뢰 99)
- 좌클릭: 칸 열기 (첫 클릭은 항상 안전 — 첫 클릭 이후 지뢰 배치)
- 우클릭: 깃발 표시/해제 (컨텍스트 메뉴는 preventDefault)
- 빈 칸(인접 지뢰 0) 오픈 시 인접 칸 자동 확장(flood fill)
- 승리 조건: 지뢰를 제외한 모든 칸을 열었을 때
- 패배 조건: 지뢰 칸을 열었을 때 → 전체 지뢰 공개
- 타이머: 첫 클릭 시 시작, 게임 종료 시 정지
- 남은 지뢰 수 카운터 (전체 지뢰 - 깃발 수)

### 편의 기능
- 난이도별 최고 기록(최단 시간) `localStorage` 저장 및 표시
- 다크모드 토글 (localStorage에 설정 저장)
- 반응형 레이아웃 (모바일에서도 보드가 화면 안에 들어오도록 셀 크기 자동 조정)

## 코딩 컨벤션
- 들여쓰기 2칸, 세미콜론 사용
- 모듈은 ES Module(`import`/`export`)로 분리, 전역 변수 최소화
- 상태(게임판, 게임 상태)와 렌더링(DOM 업데이트) 로직을 분리
- 주석은 "왜"가 비직관적인 경우에만 작성 (동작 설명 주석 지양)

## 실행 방법
빌드 과정 없음. 로컬 정적 서버로 실행 (ES Module `fetch` 정책상 `file://` 직접 열기는 브라우저에 따라 제한될 수 있음):
```
npx serve .
# 또는
python -m http.server 8080
```

## 테스트 방식
자동화 테스트 프레임워크 없음. 기능 구현 후 브라우저에서 직접 플레이하며 확인:
- 첫 클릭 안전성, flood fill, 깃발, 승리/패배, 타이머, 최고 기록 저장, 난이도 전환, 다크모드
